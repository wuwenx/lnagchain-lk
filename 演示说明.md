# 飞书 + LangChain 智能助手 — 演示说明

> 面向演示者：演示前快速过一遍，便于现场讲解与答疑。

---

## 一、项目一句话介绍

**在飞书里通过 @ 机器人对话，用 LangChain/LangGraph 做路由与技能调度，支持多轮对话、文档检索、交易所数据、定时推送与 GitLab MR 通知。**

---

## 二、演示前准备（5 分钟）

### 2.1 环境与依赖

- Python 3.10+，建议用虚拟环境：`python -m venv .venv` → 激活后 `pip install -r requirements.txt`
- 根目录 `.env` 已配置：飞书 App ID/Secret、Verification Token、OpenAI/DeepSeek 等（**不要投屏 .env 内容**）

### 2.2 启动服务

```bash
# 在项目根目录
.venv/bin/python main_webhook.py
```

- 默认：`http://0.0.0.0:9000`，飞书事件订阅的「请求地址」需指向该服务的公网 URL（如 Nginx 反代 `https://你的域名/` 或 `/webhook`，与 `WEBHOOK_PATH` 一致）。

### 2.3 飞书侧检查

- 应用已发布/可用，事件订阅里「请求地址」可访问，`FEISHU_VERIFICATION_TOKEN` 与后台一致。
- 演示用群聊中已添加该机器人，并确认「群聊中 @ 机器人」会触发（若配置了 `FEISHU_BOT_OPEN_ID`，只有 @ 到该机器人时才回复）。

## 三、推荐演示流程（约 10–15 分钟）

### 3.1 开场（1 分钟）

- **是什么**：飞书里的 AI 助手，支持自然语言 + 固定技能命令。
- **技术栈**：飞书开放平台（Webhook）、FastAPI、LangChain / LangGraph、可选 OpenAI/DeepSeek。

### 3.2 核心能力演示

| 顺序 | 演示点 | 操作 | 说明 |
|------|--------|------|------|
| 1 | **多轮对话** | 在群里 @ 机器人，先问「什么是资金费率？」，再问「用简单例子说明」 | 展示按会话（chat_id）维护历史，走 LangChain Agent |
| 2 | **技能命令** | 发送「/help」 | 列出所有已注册技能（search_doc、btc、rank、抓取、new_doc、funding_rate、jks 等） |
| 3 | **文档检索** | 「/search_doc 某某关键词」或「搜索文档 xxx」 | 命中 search_doc skill，可结合飞书文档/知识库（若已配置） |
| 4 | **抓取网页** | 「/抓取 https://某网址」 | 触发 fetch skill，抓取页面内容并总结/问答 |
| 5 | **交易所/数据** | 「/rank」或「/btc」或「/funding_rate XXX」 | 展示接交易所/CMC 等数据的能力（需对应 API 配置） |
| 6 | **Jenkins 发板** | 「/jks」或配置的触发命令 | 若有 JKS 配置，可演示一键触发构建 |

### 3.3 自动化与集成（可选，视时间）

- **定时任务**：Toobit 24h、插针告警、Popfun 日志等会按配置推送到指定飞书群；可在对应群展示已推送的卡片或说明「每 5/15 分钟自动跑一次」。
- **GitLab**：MR 创建/更新/合并、Push、Tag 会触发 Webhook，在 `FEISHU_MR_CHAT_ID` 的群收到飞书卡片；可简要说明「开发流程与飞书打通」。

### 3.4 收尾

- **架构一句话**：飞书事件 → FastAPI Webhook → 解析消息 → LangGraph 路由（fetch / skill / agent）→ 调用 LLM 或具体 skill → 回复飞书（文本/卡片）。
- 可提一句：技能可扩展，新增一个 `Skill` 实现并 `register` 即可。

## 四、可能被问到的问题（FAQ）

- **Q：支持私聊吗？**  
  A：当前逻辑主要针对群聊（含 @ 机器人）；私聊取决于飞书事件是否下发，一般需在应用权限里开启「单聊」并处理 `message` 事件。

- **Q：对话历史存哪里？**  
  A：目前是进程内按 `chat_id` 维护最近 N 轮（见 `handlers.py` 的 `MAX_HISTORY_TURNS`），重启清空；若要持久化可改为 Redis/DB。

- **Q：用的什么模型？**  
  A：`config.py` 里 `OPENAI_MODEL`（如 gpt-4o-mini、deepseek-chat），`OPENAI_API_BASE` 可改为国内中转或 DeepSeek 等兼容接口。

- **Q：如何加新技能？**  
  A：在 `skills/` 下实现符合 `Skill` 协议的模块，在 `skills/__init__.py` 的 `_register_builtin_skills()` 里 `register(xxx_skill)`，并保证在路由前被导入。

- **Q：Webhook 和 WebSocket 区别？**  
  A：本项目主入口是 `main_webhook.py`（HTTP 请求地址模式）；`main.py` 为 WebSocket 模式，适合不能暴露公网 URL 的场景，演示通常用 Webhook 即可。

## 五、关键文件速查

| 文件 | 作用 |
|------|------|
| `main_webhook.py` | Webhook HTTP 服务入口、飞书事件路由、定时任务调度 |
| `handlers.py` | 消息解析、是否回复、多轮历史、调用 LangGraph 并回写飞书 |
| `langgraph_app.py` | LangGraph 图：路由 → fetch / skill / agent → 统一 state 输出 |
| `langchain_agent.py` | 通用对话 Agent（未命中 skill 时） |
| `skills/` | 各技能实现（help、search_doc、fetch、btc、rank、new_doc、funding_rate、jks） |
| `config.py` | 从 .env 读取飞书、LLM、各 FEISHU_*_CHAT_ID 等 |
| `gitlab_webhook.py` | GitLab Webhook → 飞书 MR/Push 卡片 |
| `tasks/` | 定时任务：Toobit 24h、needle_scan、popfun_log、交易所公告等 |
